# Guia de Contribuicao

1. Crie uma branch a partir de `main`.
2. Mantenha commits pequenos e com mensagens objetivas.
3. Garanta que backend e frontend compilam antes de abrir PR.
4. Documente alteracoes relevantes no README ou em `docs/`.
5. Abra Pull Request descrevendo objetivo, escopo e evidencias de teste.

## Convencao recomendada de commits
- `feat`: nova funcionalidade
- `fix`: correcao de bug
- `refactor`: melhoria sem alterar regra
- `docs`: atualizacao de documentacao
- `chore`: tarefas de infraestrutura/build
